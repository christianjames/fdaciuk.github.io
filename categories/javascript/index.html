Como criar componentes JS usando módulos - AMD, CommonJS e UMD

PEGAR IDEIAS DAQUI

Depois de um tempo escrevendo código Javascript, você começa a perceber que algumas coisas começam a se repetir, então você pensa:

Como eu posso modularizar isso, para que esse componente seja reutilizado em vários projetos diferentes?

Para responder a essa pergunta, entram em cena duas especificações de módulo: AMD e CommonJS (ou CJS).

Eles permitem que você escreva seus códigos de forma modular, seguindo algumas definições. Vamos falar sobre cada um deles.

AMD

Asynchronous Module Definition (AMD) ficou bastante conhecido por causa do RequireJS. O formato de utilização do AMD é o seguinte:

Essa é a estrutura mais básica de um módulo escrito no formato AMD. A função define é uma global do AMD, e será usada para todo novo módulo que você criar.

Como primeiro parâmetro dessa função, você pode passar o nome do módulo - opcional - (que será usado para você fazer o “include” dele em outro lugar).

O segundo parâmetro é um array de dependências desse módulo. No nosso exemplo, colocamos como dependência a biblioteca jQuery. Esse método é chamado DI, ou Dependency Injection. ## PAREI AQUI ##

E o terceiro é a função de callback, que será chamada assim que todos os módulos do array no segundo parãmetro forem baixados, e estiverem prontos para uso.
